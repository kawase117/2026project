## Phase 6A: 短期実装（2-3時間）

### 1. データ定義の再検証 — G数閾値追加

**改善内容（ユーザー提案による）:**

```
現在の定義:
  差枚 >= 1000 → 高設定と判定

問題:
  - ランダムな大当たりで差枚が増える
  - サンプルが多すぎて、ノイズの影響が大きい

改善案:
  差枚 >= 1000 AND games >= 2000
```

**根拠:**
- G数 >= 2000 = 複数大当たり必要（ランダムな運を排除）
- サンプルは中程度（統計的に有意）
- ホール側の高設定投入位置との相関が高まる可能性

**検証方法:**
```python
# 3ホール別に統計分布を可視化
# - 差枚分布を確認
# - 「games >= 2000」フィルタの効果を測定
# - 実際の高設定台と予測の一致度を確認
```

### 2. 時系列 Cross-Validation 導入（**最重要**）

**理由（ユーザーの認識に基づく）:**
> 「過去の実績（時系列）が未来に大きな影響を与える」

**現在の問題:**
```
日付でランダム分割
├── 訓練と テストが時間的に混在
├── 時間的漏洩（Future Data Leakage）が発生
└── 未来の予測力が正確に測定されない
```

**改善:**
```
新しい時系列CV:
├── 訓練セット: 2025-01-01 ～ 2026-02-01（2ヶ月）
├── テストセット: 2026-02-01 ～ 2026-04-26（最新2.5ヶ月）
└── 実装: sklearn.model_selection.TimeSeriesSplit
```

**効果:**
- ホール側の戦略変化を正確に反映
- 実際の予測性能が見える
- 過度に楽観的なAUCを避ける

## G数閾値の比較評価

| G数閾値 | サンプル数 | 差枚分布 | 運の影響排除度 | 推奨度 |
|--------|----------|---------|--------------|--------|
| >= 1000 | 多数 | 広い | 低 | △ |
| >= 2000 | 中程度 | 絞られる | 中 | ⭐ 推奨 |
| >= 3000 | 少ない | 限定的 | 高 | △ |

**決定:** **G数 >= 2000** を実装
