# クロスメトリック検証の計算方法説明書

**作成日:** 2026-04-23  
**対象:** Pachinko Analyzer Phase 3 - ダッシュボード分析機能  
**目的:** 訓練期間のパフォーマンス指標が検証期間のコイン差に予測力を持つかを検証する手法の詳細説明

---

## 1. 概要

### 1.1 クロスメトリック検証とは

クロスメトリック検証は、異なる時期・異なるメトリクス間の相関性を測定する分析手法です。本システムでは以下の質問に答えるために用いられます：

**「訓練期間（6月/3月/1月）に勝率が高い・ゲーム数が多い属性（日付末尾、曜日、台番号末尾など）は、検証期間（4月1-20日）でも高いコイン差を記録するか？」**

### 1.2 なぜこの分析が必要か

パチンコ・パチスロホールのデータ分析では、以下の課題があります：

- **短期ノイズ**: 1日や1週間のデータは偶然性が大きい
- **季節性**: 月ごとの営業戦略やプレイヤー行動が異なる
- **因果関係の曖昧性**: 勝率が高い時期とコイン差の関係が明確でない

クロスメトリック検証により、複数の訓練期間データから派生した予測が、異なる検証期間で再現性があるか定量的に測定できます。

### 1.3 検証の基本構造

```
訓練期間データ群
├─ 6月（2025-10-01 ～ 2026-03-31）
├─ 3月（2026-01-01 ～ 2026-03-31）
└─ 1月（2026-03-01 ～ 2026-03-31）
       ↓
   メトリクス分析（勝率またはゲーム数）
       ↓
   属性グループ化（末尾数値、曜日など）
       ↓
   相対パフォーマンス計算
       ↓
   グループランキング決定
       ↓
検証期間（2026-04-01 ～ 2026-04-20）でコイン差を確認
       ↓
   一貫性スコア算出
```

## 2. 相対パフォーマンス分析の基礎概念

### 2.1 相対パフォーマンスとは

**相対パフォーマンス** = 特定グループの平均値 - 対象となるすべてのグループの全体平均値

これは、「グループの絶対値」ではなく「他のグループと比較した相対的な強さ」を表します。

#### 例：日付末尾別の勝率（6月訓練期間）

全台の平均勝率が 48% だとしましょう。

| 日付末尾 | グループ平均勝率 | 相対パフォーマンス |
|---------|-----------------|------------------|
| 0       | 51%            | 51% - 48% = +3pp |
| 1       | 49%            | 49% - 48% = +1pp |
| 2       | 47%            | 47% - 48% = -1pp |
| 3       | 48%            | 48% - 48% = 0pp  |
| 4       | 46%            | 46% - 48% = -2pp |
| ...     | ...            | ...              |
| 9       | 50%            | 50% - 48% = +2pp |

ここで「末尾0は末尾2に比べて+4pp有利」という相対的な関係が見える。

### 2.2 相対値と絶対値の違い

相対パフォーマンスを使う理由：

- **絶対値の問題**：全体的な景気変動に左右される
  - 「6月全体で勝率50%、3月全体で勝率45%」の場合、グループの絶対値を比較すると季節効果が混入
  
- **相対値の利点**：グループ間の相対的な優位性を抽出
  - 季節効果を除去し、「どの属性が他より良いか」に焦点を当てる

### 2.3 複数期間での相対パフォーマンス

3つの訓練期間（6月、3月、1月）それぞれで相対パフォーマンスを計算します。

**例：日付末尾0のゲーム数相対パフォーマンス**

```
6月訓練期間：
  末尾0のゲーム数平均 = 120G
  全体平均ゲーム数   = 110G
  相対値(6月)        = 120 - 110 = +10G

3月訓練期間：
  末尾0のゲーム数平均 = 125G
  全体平均ゲーム数   = 115G
  相対値(3月)        = 125 - 115 = +10G

1月訓練期間：
  末尾0のゲーム数平均 = 118G
  全体平均ゲーム数   = 108G
  相対値(1月)        = 118 - 108 = +10G
```

この場合、末尾0は「3期間とも一貫して+10G前後の有利」を示しています。

## 3. クロスメトリック検証の詳細プロセス

### 3.1 訓練期間と検証期間の定義

**訓練期間群:**

| 名称   | 期間範囲                  | 用途 |
|--------|---------------------------|------|
| 6月    | 2025-10-01 ～ 2026-03-31 | 長期トレンド分析 |
| 3月    | 2026-01-01 ～ 2026-03-31 | 直近3ヶ月の傾向 |
| 1月    | 2026-03-01 ～ 2026-03-31 | 直近1ヶ月の特性 |

**検証期間:**

| 期間範囲             | 用途 |
|---------------------|------|
| 2026-04-01 ～ 2026-04-20 | 訓練期間の予測力を検証 |

### 3.2 パーセンタイル分割戦略

データを「上位」「中位」「下位」3層に分割し、各層の相対パフォーマンスを独立に分析します。

**パーセンタイル分割候補:**

```python
PERCENTILE_CANDIDATES = [
    (50, 0, 50),    # 上位50% vs 下位50% （分割なし）
    (45, 10, 45),   # 上位45% / 中位10% / 下位45%
    (40, 20, 40),   # 上位40% / 中位20% / 下位40%
    (36, 28, 36),   # 上位36% / 中位28% / 下位36%
    (33, 34, 33),   # 上位33% / 中位34% / 下位33%
]
```

各候補について、「訓練期間で上位グループに属する属性が、検証期間でも高いコイン差を記録したか」を検証します。

#### 例：(40, 20, 40) での分割

対象が「日付末尾」（0-9で10グループ）の場合：

1. 各グループの6月ゲーム数相対値を計算
2. 相対値で降順ソート
3. 上位4グループ（40%）、中位2グループ（20%）、下位4グループ（40%）に分割
4. 各層について検証期間でのコイン差平均を計算

### 3.3 相対パフォーマンス計算の詳細

#### ステップ1: グループ別集計（訓練期間）

```
SELECT 
  last_digit,
  COUNT(*) as machine_count,
  AVG(games_normalized) as avg_games,
  SUM(games_normalized) as total_games
FROM machine_detailed_results
WHERE date BETWEEN '2025-10-01' AND '2026-03-31'
GROUP BY last_digit
```

結果例：

| last_digit | avg_games | 全体平均との差 |
|-----------|-----------|--------------|
| 0         | 125       | +15         |
| 1         | 110       | 0           |
| 2         | 108       | -2          |
| ...       | ...       | ...         |

#### ステップ2: 全体平均の計算

```
全体平均ゲーム数 = SUM(games_normalized) / COUNT(*)
              = 1100ゲーム（例）
```

#### ステップ3: 各グループの相対値計算

```
相対パフォーマンス(末尾i) = AVG_games(末尾i) - 全体平均

相対値(末尾0) = 125 - 110 = +15G
相対値(末尾1) = 110 - 110 = 0G
相対値(末尾2) = 108 - 110 = -2G
```

### 3.4 グループランキングと勝者決定

#### ステップ1: 各訓練期間でのランキング

6月、3月、1月それぞれについて、相対パフォーマンスでグループを順序付けます。

**6月訓練期間ランキング（末尾別、ゲーム数相対値）:**

```
1位: 末尾0 (+15G)
2位: 末尾9 (+8G)
3位: 末尾7 (+3G)
...
10位: 末尾4 (-18G)
```

**3月訓練期間ランキング:**

```
1位: 末尾0 (+20G)
2位: 末尾5 (+5G)
3位: 末尾8 (+4G)
...
```

**1月訓練期間ランキング:**

```
1位: 末尾0 (+12G)
2位: 末尾9 (+6G)
3位: 末尾3 (+2G)
...
```

#### ステップ2: パーセンタイル分割による勝者決定

(40, 20, 40) パーセンタイル分割の場合（10グループ）：

- **上位層**: 4グループ（グループ全体の上位40%）
- **中位層**: 2グループ（グループ全体の中位20%）
- **下位層**: 4グループ（グループ全体の下位40%）

各訓練期間で：

**6月:**
```
上位層 (4グループ) : 末尾 [0, 9, 7, 6]
中位層 (2グループ) : 末尾 [5, 3]
下位層 (4グループ) : 末尾 [4, 2, 1, 8]
```

**3月:**
```
上位層 (4グループ) : 末尾 [0, 5, 8, 1]
中位層 (2グループ) : 末尾 [7, 3]
下位層 (4グループ) : 末尾 [4, 2, 9, 6]
```

**1月:**
```
上位層 (4グループ) : 末尾 [0, 9, 3, 7]
中位層 (2グループ) : 末尾 [5, 1]
下位層 (4グループ) : 末尾 [4, 2, 6, 8]
```

#### ステップ3: 各層の勝者を決定

各層（上位、中位、下位）について、「3期間すべてで勝者か、それとも複数期間で勝者か」を判定します。

**判定ルール:**

- **強勝者**: 3期間すべてで該当層に属する
- **弱勝者**: 2期間で該当層に属する（1期間は別層）
- **敗者**: 1期間以下で該当層に属する

上記例では、末尾0は「全3期間で上位層」なので、**上位層の強勝者**です。

### 3.5 相対パフォーマンスの統計指標（μ, σ）

3つの訓練期間から計算された相対パフォーマンス値を統計的に要約します。

#### ステップ1: グループごとの相対値を抽出

例：末尾0のゲーム数相対値

```
6月:  +15G
3月:  +20G
1月:  +12G
```

#### ステップ2: 平均（μ）を計算

```
μ(末尾0) = (15 + 20 + 12) / 3 = 15.67G
```

この値は「末尾0は平均で全体より15.67Gゲーム数が多い傾向」を意味します。

#### ステップ3: 標準偏差（σ）を計算

```
σ(末尾0) = sqrt(((15-15.67)² + (20-15.67)² + (12-15.67)²) / 3)
         = sqrt((0.45 + 18.78 + 13.45) / 3)
         = sqrt(10.89)
         = 3.30G
```

σが小さい = 3期間で一貫して似た値（予測性が高い）  
σが大きい = 3期間でばらつき（予測性が低い）

### 3.6 一貫性スコア計算

**一貫性スコア** = グループが3期間すべてで同じパーセンタイル層に属しているか否か

#### 定義

```
一貫性スコア = {
  3   : 3期間すべてで同じ層（上位/中位/下位）に属する
  2   : 2期間で同じ層に属する
  1   : 1期間のみ該当層に属する
  0   : 0期間で該当層に属する（該当層以外）
}
```

#### 例

| グループ | 6月       | 3月       | 1月       | スコア |
|---------|----------|----------|----------|--------|
| 末尾0   | 上位層   | 上位層   | 上位層   | 3      |
| 末尾9   | 上位層   | 下位層   | 上位層   | 2      |
| 末尾4   | 下位層   | 下位層   | 下位層   | 3      |
| 末尾5   | 中位層   | 上位層   | 中位層   | 2      |

一貫性スコア3 = 予測性が高い信頼できるグループ  
一貫性スコア2 = 予測性がやや低い  
一貫性スコア0-1 = ノイズの可能性が高い

### 3.7 比率最適化

複数のパーセンタイル分割候補を検証し、「どの分割が最も強い予測力を持つか」を判定します。

#### パーセンタイル候補の評価指標

各候補 (upper_pct, middle_pct, lower_pct) について、検証期間でのコイン差を測定：

```python
# 例：(40, 20, 40) 分割の場合
PERCENTILE_CANDIDATES = [
    (50, 0, 50),    # 指標: Upper層の平均コイン差
    (45, 10, 45),   # 指標: Upper層の平均コイン差
    (40, 20, 40),   # 指標: Upper層の平均コイン差
    (36, 28, 36),   # 指標: Upper層の平均コイン差
    (33, 34, 33),   # 指標: Upper層の平均コイン差
]
```

#### 評価ロジック

各候補について：

1. 訓練期間でパーセンタイル分割を実施
2. 上位層（Upper）に属するグループを抽出
3. 検証期間でそれらグループのコイン差平均を計算
4. 候補ごとに比較して、最高値を達成した分割を「最適」とする

**例**

```
(50, 0, 50) : Upper層のコイン差平均 = +245枚
(45, 10, 45) : Upper層のコイン差平均 = +258枚  ← 最適
(40, 20, 40) : Upper層のコイン差平均 = +235枚
(36, 28, 36) : Upper層のコイン差平均 = +220枚
(33, 34, 33) : Upper層のコイン差平均 = +205枚
```

この場合、**(45, 10, 45) 分割が最も強い予測力を示した**と結論付けます。

## 4. 統計指標の解釈

### 4.1 相対値μ（平均相対パフォーマンス）

**定義:**

```
μ = (相対値_6月 + 相対値_3月 + 相対値_1月) / 3
```

**解釈:**

- μ > 0 : グループ全体より有利（平均して勝ちやすい）
- μ = 0 : グループ全体と同等
- μ < 0 : グループ全体より不利（平均して負けやすい）

**例：末尾0のゲーム数**

```
μ = +15.67G

解釈：末尾0は「他の末尾より平均で15.67Gゲーム数が多い傾向」
→ 営業側が末尾0の台にゲーム数を多くセットする傾向が見られる
```

### 4.2 相対値σ（標準偏差）

**定義:**

```
σ = sqrt(Σ(相対値_i - μ)² / 3)
```

**解釈:**

- σが小さい（例：2G） : 3期間でほぼ一貫した値 → 再現性が高い
- σが大きい（例：8G） : 3期間でばらつき → 一時的なノイズの可能性

**例：2つのグループの比較**

```
末尾0:
  μ = +15G, σ = 2G
  → 「ほぼ確実に+15G前後の有利」（再現性高）

末尾5:
  μ = +8G, σ = 7G
  → 「+1G～+15Gと大きなばらつき」（再現性低）

結論：末尾0の方が「勝者のグループ」として信頼できる
```

### 4.3 一貫性スコア

**スコア分布と解釈:**

```
スコア3（一貫性高）:
  ├─ グループの特性が安定している
  ├─ 訓練期間全体で同じ層に属する
  └─ 検証期間での予測性が高い

スコア2（一貫性中）:
  ├─ 2期間では一貫した層
  ├─ 1期間で層が異なる
  └─ 検証期間での予測性は中程度

スコア0-1（一貫性低）:
  ├─ 複数の層に散らばっている
  ├─ 偶然性やノイズが影響している可能性
  └─ 検証期間での予測性が低い
```

**例：複数グループの比較**

```
  グループ   スコア  μ(ゲーム数)  σ   検証期間コイン差
  末尾0      3      +15G        2G   +256枚
  末尾9      2      +6G         4G   +120枚
  末尾4      3      -14G        3G   -198枚
  末尾5      1      +2G         6G   +45枚

解釈：
- 末尾0：スコア3、μ正、σ小 → 最も信頼できる勝者グループ
- 末尾4：スコア3、μ負、σ小 → 最も信頼できる敗者グループ
- 末尾5：スコア1、μ小、σ大 → 予測不可能なノイズ
```

### 5.3 一貫性スコア計算

各グループが3期間で上位/中位/下位層のどこに属したかを整理：

```
グループ   6月上位/中位/下位   3月上位/中位/下位   1月上位/中位/下位
末尾0      上位              上位              上位        → スコア3
末尾1      中位              中位              中位        → スコア3
末尾2      下位              下位              下位        → スコア3
末尾3      中位              中位              中位        → スコア3
末尾4      下位              下位              下位        → スコア3
末尾5      上位              上位              上位        → スコア3
末尾6      下位              下位              下位        → スコア3
末尾7      上位              上位              上位        → スコア3
末尾8      下位              下位              下位        → スコア3
末尾9      上位              上位              上位        → スコア3
```

この例では「全グループでスコア3（完全一貫）」という理想的な結果です。

### 5.4 統計指標（μ, σ）計算

```
末尾0（上位層）:
  相対値群 = [+15G, +20G, +12G]
  μ = (+15 + 20 + 12) / 3 = +15.67G
  σ = sqrt(((15-15.67)² + (20-15.67)² + (12-15.67)²) / 3)
    = sqrt(10.89) = 3.30G
  → 小さいσで一貫性高

末尾4（下位層）:
  相対値群 = [-18G, -20G, -20G]
  μ = (-18 - 20 - 20) / 3 = -19.33G
  σ = sqrt(((-18+19.33)² + (-20+19.33)² + (-20+19.33)²) / 3)
    = sqrt(0.63) = 0.79G
  → 非常に小さいσ、一貫性最高

末尾5（上位層）:
  相対値群 = [+5G, +5G, +4G]
  μ = (+5 + 5 + 4) / 3 = +4.67G
  σ = sqrt(((5-4.67)² + (5-4.67)² + (4-4.67)²) / 3)
    = sqrt(0.22) = 0.47G
  → 最小のσ
```

### 5.5 検証期間でのコイン差測定

検証期間（2026-04-01～2026-04-20）でのグループ別コイン差結果：

```
グループ   訓練期間層   一貫性   μ(相対値)   σ    検証期間コイン差
末尾0      上位        3      +15.67G    3.3G  +265枚
末尾5      上位        3      +4.67G     0.47G +156枚

末尾9      上位        3      +8.67G     1.53G +210枚

末尾1      中位        3      -3.33G     1.15G +32枚
末尾3      中位        3      +2.0G      0.0G  +78枚

末尾2      下位        3      -6.0G      0.58G -145枚

```

### 5.6 検証結果と仮説の有効性

**観察:**

1. **上位層の勝率**: 4グループすべてが正のコイン差を記録
2. **下位層の敗率**: 4グループすべてが負のコイン差を記録
3. **中位層の不確実性**: グループごとに+32～+78で、中位層らしく中程度

**結論:**

```
仮説A（訓練期間の勝者は検証期間でも勝つ）：有効性確認
  - 上位層4グループの平均コイン差: (+265 + 156 + 98 + 210) / 4 = +182.25枚
  - 下位層4グループの平均コイン差: (-145 - 312 - 210 - 175) / 4 = -210.5枚
  - 差分: +182.25 - (-210.5) = +392.75枚 ← 明確な優位性

勝率/ゲーム数の訓練期間パフォーマンスは、
検証期間のコイン差に対して強い予測力を持つことが実証された
```

## 6. 重要な注意事項と制限

### 6.1 データの質と量の要件

**最小データ量:**

- 各訓練期間で各グループ最低10日以上必要
- グループ内の台数が少ないと統計信頼性が低下

**外れ値の処理:**

- 極端に高い/低い日を除外する必要がある場合がある
- 営業停止日やイベント日の影響を考慮

### 6.2 季節性とカレンダー効果

**注意点:**

- 6月訓練期間（10月～3月）は秋冬季
- 3月訓練期間（1月～3月）は冬春季
- 検証期間（4月1-20日）は春季初期

季節による本質的な違いが結果に影響する可能性あり。

**対策:**

- 同季節のデータ間での比較を優先
- 季節効果を明示的にモデル化する検討

### 6.3 統計的有意性の限界

クロスメトリック検証は「相関性」を測定するが、「因果性」を証明しない。

```
例：末尾0が勝者だから営業側が末尾0に高いゲーム数をセットした
  ↔ 営業側が末尾0にゲーム数を多くセットしたから末尾0が勝った

双方向の関係が考えられる
```

### 6.4 比率最適化の過学習リスク

複数のパーセンタイル候補を試す際、「検証期間に対して最適なパーセンタイル」が結果として選ばれる。

これは**事後的な最適化**であり、来月以降のデータでは同じパーセンタイルが最適かは不明。

**対策:**

- 複数期間での検証を繰り返す
- スコア3（一貫性高）のグループに重点を置く

### 6.5 自動解釈の落とし穴

相対パフォーマンスが高い = 「そのグループは絶対値でも高い」ではない。

```
例：
  全体平均勝率が低い時期（45%）: グループ平均50% → 相対値+5%
  全体平均勝率が高い時期（55%）: グループ平均56% → 相対値+1%

相対値は低いが、絶対値での勝率は56% > 50%（実は高い）
```

## 7. 拡張可能性と今後の応用

### 7.1 複合属性検証

現在は単一属性（日付末尾、曜日など）での分析だが、複合属性への拡張が可能：

```
検証対象の例：
- 日付末尾 AND 曜日 の組み合わせ
- 台番号末尾 AND 機種タイプ の組み合わせ
- 日付末尾 AND 営業戦略コード の組み合わせ
```

### 7.2 時系列モデル化

複数のクロスメトリック検証を時系列で実施し、「パーセンタイル最適値の推移」をモデル化：

```
2月検証 → 最適パーセンタイル: (40, 20, 40)
3月検証 → 最適パーセンタイル: (45, 10, 45)
4月検証 → 最適パーセンタイル: (50, 0, 50)

傾向：中位層の重要性が低下している
→ 営業戦略の変化を反映している可能性
```

### 7.3 メトリクス間の相互作用分析

現在は「勝率」または「ゲーム数」単体だが、相互作用を考慮：

```
勝率が高い AND ゲーム数も多い グループは？
  → 「最高のグループ」として重み付け

勝率が高いが ゲーム数が少ない グループは？
  → 「サンプル数不足による見かけの勝利」の可能性
```

### 7.4 機械学習への統合

クロスメトリック検証の結果を特徴量として機械学習モデルに入力：

```
入力特徴量：
  - μ値 (平均相対パフォーマンス)
  - σ値 (標準偏差)
  - 一貫性スコア
  - パーセンタイル層 (上位/中位/下位)

予測対象：
  - 翌月のコイン差
  - グループの順位変動

モデル：勾配ブースティング、ニューラルネットワークなど
```

### 7.5 リアルタイム更新

バッチ処理ではなく、新しい検証期間データが入るたびに自動更新：

```
現在：月1回のバッチ実行
改善案：
  - 週1回の更新
  - 日単位の自動更新
  - リアルタイムスコア表示ダッシュボード
```

## 8. 用語集

| 用語 | 定義 |
|------|------|
| **クロスメトリック検証** | 異なる時期・異なるメトリクス間の予測力を測定 |
| **訓練期間** | 過去の長期データ（6月/3月/1月） |
| **検証期間** | 検証対象の短期データ（4月1-20日） |
| **相対パフォーマンス** | グループ平均 - 全体平均 |
| **パーセンタイル分割** | データを複数層に分割（上位/中位/下位） |
| **μ（ミュー）** | 相対パフォーマンスの平均値 |
| **σ（シグマ）** | 相対パフォーマンスの標準偏差 |
| **一貫性スコア** | 3期間で同層に属する度合い（0-3） |
| **勝者グループ** | 訓練期間で上位層に属するグループ |
| **敗者グループ** | 訓練期間で下位層に属するグループ |
| **比率最適化** | 複数パーセンタイルから最適なものを選定 |
