# バックテスト検証ページ 設計書

**作成日**: 2026-04-23
**ファイル**: `dashboard/pages/page_15_backtest_validation.py`

## 目的

過去（1月～3月）の実績データから算出したランキングが、4月の同条件で再現するかを検証する。ホールごとに戦略の有効性を定量的に評価する。

---

## 分析パターン

6つの固定パターンを全て検証する（選択式ではない）：

| 軸1（日付属性）              | 軸2（台属性） |
|---------------------------|------------|
| DD別（月内日付：1日～31日）  | 台末尾別    |
| DD別（月内日付：1日～31日）  | 台番号別    |
| DD別（月内日付：1日～31日）  | 機種別      |
| 曜日別（月曜～日曜）         | 台末尾別    |
| 曜日別（月曜～日曜）         | 台番号別    |
| 曜日別（月曜～日曜）         | 機種別      |

---

## 検証期間

- **訓練期間**: 2026-01-01 ～ 2026-03-31（3か月）
- **テスト期間**: 2026-04-01 ～ 2026-04-20（20日間）

将来的にスライディング窓方式（ウォークフォワード）への拡張を検討するが、現フェーズでは固定期間のみ実装。

---

## ランキング指標

各パターンごとに以下3指標でランキングを計算する：

| 指標     | 内容               |
|---------|-------------------|
| 勝率     | 差枚が正の台の割合   |
| 平均差枚 | 期間中の台平均差枚   |
| 平均G数  | 期間中の台平均G数    |

### ランク判定基準

TOP100ではなく **TOP20% / TOP10%** を採用。  
理由：台末尾（11種）・台番号（数百）・機種（数十）で母数が大きく異なるため、固定件数では不公平になる。

---

## 検証指標

4月の毎日について、以下を判定する：

| 指標名       | 判定内容                                           |
|------------|--------------------------------------------------|
| ランク維持度  | 過去TOP20%/10%に入っていたパターンが当日も同水準に入るか |
| 利益性       | 当日の勝率が正の値か（差枚がプラスか）                 |
| TOP20%維持率 | 20日間でTOP20%を維持した日数の割合（%）               |
| TOP10%維持率 | 20日間でTOP10%を維持した日数の割合（%）               |
| 利益性維持率  | 20日間で勝率が正だった日数の割合（%）                 |

---

## ページレイアウト

6パターン分のセクションを縦に並べる（同一レイアウトを6回繰り返す）。

```
[ページタイトル] バックテスト検証 (2026-01 ～ 2026-03 → 2026-04)

─── DD × 台末尾 ───────────────────────────────────
  [グラフA] 信頼度ランキングTOP10
            - 勝率 / 差枚 / G数 それぞれでTOP10を棒グラフ表示
  [グラフB] 指標の分布
            - 全パターンの「ランク維持率」「利益率」平均を比較
  [グラフC] ヒートマップ
            - 軸1 × 軸2 の信頼度（TOP20%維持率）をヒートマップ表示
  [テーブル] ソート・フィルタ付き詳細テーブル
            左側（固定）: 軸1 | 軸2 | 過去勝率 | 過去差枚 | 過去G数 | 過去ランク
            中央（日付）: 4/1 | 4/2 | ... | 4/20（各セルに3指標×2閾値の記号）
            右側（集計）: TOP20%維持率 | TOP10%維持率 | 利益率

─── DD × 台番号 ─── （同一構成）
─── DD × 機種 ──── （同一構成）
─── 曜日 × 台末尾 ─ （同一構成）
─── 曜日 × 台番号 ─ （同一構成）
─── 曜日 × 機種 ─── （同一構成）
```

### テーブルの各セル表示（4月毎日）

各セルに3指標×2閾値（TOP20%/TOP10%）＋利益性を記号で並列表示：

```
勝率20%:✓ 差枚20%:✓ G数20%:△
勝率10%:✗ 差枚10%:✓ 利益性:✓
```

## 計算ロジック

### Step 1：訓練期間の集計（1月～3月）

`machine_detailed_results` から各パターンごとにグループ集計：

```python
# 例: DD × 台末尾 パターン
df_train = df[(df['date'] >= '20260101') & (df['date'] <= '20260331')]
df_group = df_train.groupby(['dd', 'last_digit']).agg(
    avg_win_rate=...,
    avg_diff=...,
    avg_games=...,
    count=('date', 'count')
)
# ランク計算（パーセンタイル）
df_group['rank_pct_win'] = df_group['avg_win_rate'].rank(pct=True)
```

### Step 2：TOP20% / TOP10% リスト作成

```python
top20_win = df_group[df_group['rank_pct_win'] >= 0.80]
top10_win = df_group[df_group['rank_pct_win'] >= 0.90]
```

### Step 3：4月の毎日検証

4月1日～20日の各日について：
1. その日の実績データを読込
2. 訓練期間のTOP20%/10%リストと照合
3. 当日の勝率が正か判定

---

## エラーハンドリング・制約

| ケース                              | 対応                                   |
|-----------------------------------|--------------------------------------|
| 曜日別パターンで出現回数が5回未満    | 信頼性低として除外または薄色表示          |
| DD別パターンの出現回数フィルタ       | 適用しない（月1回のため3か月で最大3回が上限）|
| 4月に実績がない日付                | その日はスキップ（テーブルに「データなし」）|
| 空DataFrameの場合                  | 既存 `charts.py` の挙動に従い空グラフ返却 |
