# パチスロ分析データベース スキーマ説明書 **（ランク・履歴カラム完全版 + 第X曜日対応）** --- ## ⚠️ 重要な注意事項 ### is_zorome カラムの型変換に関する注意 **問題**: SQL 処理時に BOOLEAN/INTEGER 型の混同が発生しやすい **原因**: - SQLite では BOOLEAN が存在せず、INTEGER（0/1）で代用 - Python の boolean と SQL の INTEGER が相互変換される際にエラーが起きやすい - json_processor.py での計算時に bool 型が返されることがある **具体的なエラーパターン**: ❌ **NG: Boolean 型での比較** ```sql WHERE is_zorome = True # エラー或いは予期しない結果 ``` ✅ **OK: INTEGER での比較** ```sql WHERE is_zorome = 1 # 正確 ``` ❌ **NG: Python での処理** ```python if row['is_zorome']: # 型不一致の可能性 ``` ✅ **OK: 明示的に INTEGER に変換** ```python if int(row['is_zorome']) == 1: # 安全 ``` **推奨対処**: 1. **SQL クエリ時**: `WHERE is_zorome = 1` または `WHERE is_zorome = 0` で判定 2. **Python での処理**: `int(is_zorome)` で明示的に INTEGER に変換 3. **DB 投入時**: `json_processor.py` の `calculate_last_digit_and_zorome()` で常に INTEGER を返すことを確認 4. **テスト時**: is_zorome カラムが確実に 0 または 1 であることを検証 **影響を受けるモジュール**: - `json_processor.py`: `calculate_last_digit_and_zorome()` 関数 - `summary_calculator.py`: ゾロ目フィルタ処理 - `database_accessor.py`: 条件判定 - Notebook での分析時 --- ##
